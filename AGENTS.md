# AGENTS.md

## Role
You are helping build a professional developer portfolio using Next.js, TypeScript, Tailwind CSS, Motion, and Vercel.

## Main Objective
Build a clean, animated, responsive, and portfolio-ready website for a software engineering / full-stack developer profile.

## Required Behavior
Always read and follow DESIGN.md before changing UI or layout.

## Coding Rules
- Use TypeScript.
- Use reusable React components.
- Use Tailwind CSS consistently.
- Keep folder structure organized.
- Avoid unnecessary dependencies.
- Use semantic HTML where possible.
- Keep components readable and maintainable.

## Animation Rules
Use Motion for animation.

Allowed animation:
- Fade-up section reveal
- Hero text stagger
- Button hover animation
- Card hover lift
- Subtle floating background shapes
- Smooth transition between states

Do not use:
- Excessive animation
- Random bouncing effects
- Distracting motion
- Heavy animation that affects performance

## Content Rules
The portfolio should emphasize:
- Full-stack web application development
- Dashboard systems
- CRUD features
- Database integration
- Clean project structure
- Practical software engineering implementation

## Featured Project
EduManage / Course Management System should be the main featured project.

Present it as a full-stack academic/course management system with:
- Dashboard interface
- CRUD features
- Database integration
- Authentication / role system only if available
- Clean UI structure

## Do Not
- Do not invent fake company experience.
- Do not exaggerate the user's skill level.
- Do not claim production use unless provided.
- Do not replace the design system without instruction.
